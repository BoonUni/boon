บุญฟอนต์

บุญ คือ ชุดตัวอักษรตระกูลหนึ่ง (font family) ที่มุ่งแสดงผลตัวอักษรไทยให้ถูกต้องในเว็บบราวเซอร์ ทุกตัวอักษรทั้งไทยและละตินนั้นวาดขึ้นมาใหม่โดยตั้งใจให้สามารถใช้งานเป็นฟอนต์ไทยตัวอ่านมีหัวที่พอดูได้ในขนาด 14px ขึ้นไป และมีความหนาใกล้เคียงกับตัวละติน

ดาวน์โหลดบุญ ๛

๏ สารบัญ

  • แรงบันดาลใจ
  • สัญญาอนุญาต
  • วิธีใช้งาน
  • กรณีทดสอบ

๏ แรงบันดาลใจ

บุญฟอนต์ ได้รับแรงบันดาลใจจากฟอนต์อื่นหลายตระกูล เช่น Roboto, Helvetica (Arial), Thonburi, Garuda และ SP Laksaman (หรือ TH Sarabun PSK ที่ปรับสเกลใหม่)

ตัวอักษรไทยใน บุญฟอนต์ จะเล็กกว่าฟอนต์ไทยทั่วไปเมื่อจับคู่กับตัวละติน (อย่างน้อยก็ฟอนต์ไทยใน Linux ที่ผมใช้อยู่) โดยตั้งใจกำหนด ความสูง บ. ให้ต่ำกว่า x-height เล็กน้อย เพราะการเขียนคำไทยนั้นใช้พื้นที่ในแนวตั้งมากกว่าคำภาษาอังกฤษ เพื่อให้ใช้งานฟอนต์ไทยในเว็บได้สบายตาโดยไม่ต้องกำหนด line-height เกิน 1.5em ใน CSS โดยที่ตัวไทยไม่แน่นเกินไปและตัวละตินไม่ห่างเกินไปในแนวตั้ง ถ้าเข้าใจไม่ผิด ฟอนต์ไทยของ iOS (Thonburi?) ก็แก้ปัญหาแบบนี้ โดยให้ สูง บ. ต่ำกว่า x-height ของ Helvetica Neue เล็กน้อย นอกจากนี้บุญฟอนต์ยังเพิ่มความหนาของตัวอักษรไทยให้ใกล้เคียงกับตัวละติน บุญตัวหนา จึงหนามากเมื่อเทียบกับฟอนต์ไทยตัวอื่นแต่ดูกลมกลืมเมื่อใช้คู่กับตัวละติน (ถ้าผมไม่คิดไปเองนะ)

บุญฟอนต์ เพิ่งจะเริ่มต้นพัฒนาและทดลองคุณสมบัติต่างๆ (และโทษสมบัติ) ยังห่างไกลจากคำว่าสมบูรณ์แบบ หวังพึ่ง Anti-Alias หรือ Text Renderer ในระบบปฏิบัติการเท่านั้น เพราะไม่ได้ Hint หรือทำ Grid Fit ตัวหนา (Bold) ยัง kern ได้แย่และเบียดกันมากเมื่อใช้ในขนาดเล็ก ส่วน ตัวเอน (Italic) ก็ใช้โปรแกรมปรับอัตโนมัติมันจึงดูขาดๆ เกินๆ และอาจมีการเปลี่ยนแปลงคุณสมบัติอีกหลายอย่างได้ทุกเมื่อทุกเวลาเมื่อทดสอบการใช้งานในหลายระบบปฏิบัติการและหลายเว็บบราวเซอร์ซึ่งมีความสามารถในการรองรับ OpenType Features ไม่เท่ากัน โดยเฉพาะอย่างยิ่งสำหรับตัวอักษรยูนิโค้ดที่ไม่ใช่ละตินอย่างอักษรไทย

๏ สัญญาอนุญาต

บุญฟอนต์ ใช้สัญญาอนุญาต (license) แบบ GNU GPLv3 (+Font Exception) แปลว่าคุณมีอิสระเต็มในการใช้งาน ดัดแปลง หรือปรับปรุง โดยไม่ต้องขออนุญาตเจ้าของลิขสิทธิ์ (จะทำอะไรกับมันก็ไม่ต้องบอกผมก่อน) ข้อแม้คือเมื่อดัดแปลงจากต้นแบบแล้วอยากจะเผยแผ่ผลงานใหม่ก็ต้องใช้สัญญาอนุญาตแบบเดียวกันและต้องเปิดเผยซอร์สโค้ดเช่นกัน ส่วนคำว่า +Font Exception ที่ต่อท้ายมานั้นเพื่อบอกผู้ใช้ที่ต้องการฝังฟอนต์ในเอกสารว่าไม่ต้องกังวล เพราะระบุไว้ชัดเจนว่าเอกสารที่เกิดจากการใช้งานฟอนต์นี้ (เช่น html page, pdf, etc.) ไม่จำเป็นต้องใช้สัญญาอนุญาตแบบเดียวกับตัวฟอนต์

บุญฟอนต์ แจกจ่ายและเปิดเผยซอร์สโค้ดด้วยหวังว่ามันจะมีประโยชน์และเป็นส่วนหนึ่งของการพัฒนาฟอนต์ไทยเพื่อให้เหมาะต่อการใช้งานทั้งในโปรแกรมทั่วไปและในเว็บไซต์ และโปรดทราบไว้ล่วงหน้าว่าคุณต้องยอมรับความเสี่ยงจากการใช้งานฟอนต์นี้เอง ผู้พัฒนาไม่ต้องรับผิดชอบต่อความเสียหายที่อาจจะเกิดจากการใช้งานฟอนต์นี้ ไม่ว่ากรณีใดๆ

๏ วิธีใช้งาน

คุณสามารถดาวน์โหลดไฟล์ฟอนต์และซอร์สโค้ดเวอร์ชั่นล่าสุดได้ที่ github.com หรือโคลนทั้งโปรเจ็คต์เพื่อดัดแปลงด้วยโปรแกรม FontForge ได้ที่ https://github.com/sungsit/boon

๑. การใช้งานกับโปรแกรมทั่วไป

ติดตั้งไฟล์ฟอนต์ประเภท OpenType จาก /fonts/otf/*.otf ณ เวลาที่เขียน ทดสอบใน Ubuntu 13.04 เท่านั้น ยังไม่ได้ทดสอบในระบบปฏิบัติการอื่น แต่น่าจะใช้งานได้ปกติสำหรับระบบปฏิบัติการและโปรแกรมที่รองรับ OpenType

๒. การใช้งานในเว็บไซต์

ใช้ไฟล์ฟอนต์จาก /fonts/woff/*.woff ไฟล์ .woff ก็คือ TrueType นั่นเอง แต่ห่อและบีบให้ขนาดเล็กลงเพื่อให้เหมาะต่อการใช้งานในอินเตอร์เน็ต นั่นเป็นเหตุผลที่บุญฟอนต์ไม่มีไฟล์ .ttf เพราะไฟล์ .otf กับ .woff ก็เกินพอ (แถม .svg เผื่อใครไม่ถนัดใช้ FontForge)

การใช้ฟอนต์ไทยในเว็บไซต์นั้นจำเป็นต้องใช้ CSS และ โชค เข้าช่วย โชคในที่นี้หมายความว่าเว็บบราวเซอร์แต่ละตัวรองรับ OpenType Features ไม่เท่ากัน แม้จะใช้ @font-face ก็ตาม

ณ เวลาที่เขียน แม้จะยังไม่ได้ทดสอบแต่ก็แน่ใจว่า บุญฟอนต์ ไม่สนับสนุนการใช้งานกับบราวเซอร์รุ่นเก่าหลายรุ่น ตัวอย่างเช่น Internet Explorer ต่ำกว่ารุ่น 9 และ mobile browser อีกหลายตัว เพราะถึงแม้ IE รุ่นเก่าจะสามารถใช้งาน @font-face กับไฟล์ .eot ได้ หรือบราวเซอร์ในอุปกรณ์เคลื่อนที่รุ่นเก่าจะเรียกใช้ฟอนต์จากไฟล์ .svg ได้ แต่เท่าที่ผมจำได้ไฟล์พวกนี้ไม่รองรับ opentype features นั่นเป็นเหตุผลที่วรรณยุกต์ไทยต้องลอยสูงไว้ก่อนเพราะมันเป็นตำแหน่งที่ปลอดภัยที่สุดในบราวเซอร์ที่แย่ที่สุด จริงๆ สามารถใช้งานฟอนต์ไทยทุกตัวกับ IE รุ่นเก่าได้ แต่คุณต้องแปลงฟอนต์ฟอร์แมตเองและต้องยอมรับด้วยว่าวรรณยุกต์ไทยต้องลอยอย่างหลีกเลี่ยงไม่ได้ จะได้ไม่โกรธฟอนต์หรือคนทำฟอนต์! (ผมคิดว่าสำหรับ IE รุ่นเก่านั้นแนะนำให้ผู้ใช้ติดตั้งฟอนต์ไทยเพิ่มในระบบปฏิบัติเองคงง่ายกว่าการแก้ปัญหาวิธีอื่น!)

คุณสามารถดูตัวอย่างการใช้ CSS กับบุญฟอนต์ได้จากซอร์สโค้ดของหน้าเว็บนี้ หรือจากไฟล์ /css/boon.css ที่สำคัญคืออย่าลืมใช้ text-rendering: optimizelegibility และ/หรือ font-feature-settings เพื่อบอกบราวเซอร์ที่ฉลาดพอให้เรียกใช้งานคุณสมบัติของ OpenType ที่ฟอนต์ตัวนั้นมี (smcp, kern, liga เป็นต้น) แม้บราวเซอร์รุ่นใหม่บางตัวจะเปิดไว้ให้โดยอัตโนมัติแล้วก็ตาม

การใช้เว็บฟอนต์ต้องอธิบายกันยาวทีเดียวเพราะมีรายละเอียดที่ต้องคำนึงถึงเยอะมากๆ และจะพยายามเขียนตัวอย่างสำหรับการใช้งาน CSS กับฟอนต์ไทยในอนาคต

๏ กรณีทดสอบ

หากคุณไม่เห็นคำว่า บุญฟอนต์ จาก หัวข้อด้านบนสุด แบบภาพด้านล่างนี้ ก็แสดงว่าบราวเซอร์ที่คุณใช้งานไม่รองรับ @font-face กับไฟล์ .woff ดังนั้นไม่มีประโยชน์ที่จะอ่านต่อ ยกเว้นคุณจะเปลี่ยนบราวเซอร์

๓. Kerning ตัวอักษรละติน

อักษรละตินบางตัวต้องจัดการความห่างในแนวราบระหว่างกันเป็นพิเศษ เช่น ตัว T, A, V, W, Y ใหญ่ กับ ตัวอักษรเล็กอื่นๆ อีกหลายตัว เพื่อไม่ให้มันดูห่างกันมากเกินไป เว็บบราวเซอร์ส่วนใหญ่จะไม่เรียกใช้ Kerning features จากฟอนต์จนกว่าจะระบุใน CSS ด้วย text-rendering: optimizelegibility (โดยปกติตัวอักษรไทยไม่ต้อง kern ในแนวราบก็ได้ ยกเว้นสระหน้าตัวสูงอย่าง โ ใ และ ไ เพราะพยัญชนะแต่ละตัวใช้พื้นที่ว่างค่อนข้างใกล้เคียงกัน และมีตำแหน่งสระบน / สระล่าง / วรรณยุกต์ คอยกำกับระยะห่างเส้นหลังอยู่จึงไม่มีปัญหาแบบตัวละตินเล็ก-ใหญ่)
